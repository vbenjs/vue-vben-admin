# 系统管理平台一期实施清单

本文档定义 RISS 在“系统管理平台优先”策略下的第一阶段实施范围，默认前端主应用为 `apps/web-antd`。

## 建设目标

第一阶段先把平台底座做稳，作为后续“财务管理”和“出纳管理”的前置条件。

平台一期目标：

- 跑通真实登录与权限体系
- 跑通系统管理菜单与页面
- 跑通组织、角色、用户、字典、参数等主数据
- 跑通表单设计、审批流程、打印设计三类平台能力
- 为后续财务、出纳模块提供稳定底座

## 前端选择

平台一期前端统一使用 `apps/web-antd`，原因如下：

- 当前系统管理页面在 `web-antd` 中更完整
- 可减少重复搬运页面的成本
- 能更快进入联调和数据接入阶段

## 一期范围

### A. 认证与权限

- 登录
- 登出
- 刷新 Token
- 当前用户信息
- 权限码获取
- 动态菜单加载

### B. 组织与权限主数据

- 用户管理
- 角色管理
- 菜单管理
- 部门管理
- 岗位管理
- 租户管理

### C. 平台配置能力

- 字典管理
- 参数配置
- 单据编号规则
- 定时任务
- 操作日志
- 登录日志

### D. 低代码与流程能力

- 表单设计
- 审批流程设计
- 打印模板设计
- 表单数据管理

## 建议实施顺序

### P1：认证与菜单联调

接口优先级：

- `POST /api/auth/login`
- `POST /api/auth/refresh`
- `POST /api/auth/logout`
- `GET /api/auth/codes`
- `GET /api/user/info`
- `GET /api/menu/all`

完成标准：

- `web-antd` 可脱离 mock 真实登录
- 页面刷新后菜单能正确回显
- 按角色控制菜单和按钮权限

### P2：组织权限主数据

优先模块：

1. 用户管理
2. 角色管理
3. 菜单管理
4. 部门管理
5. 岗位管理
6. 租户管理

完成标准：

- 新增、编辑、删除、分页查询可用
- 基础校验和状态切换可用
- 角色菜单授权可用

### P3：平台配置能力

优先模块：

1. 字典管理
2. 参数配置
3. 编号规则
4. 登录日志
5. 操作日志
6. 定时任务

完成标准：

- 财务模块依赖的字典、参数、编号规则可配置
- 日志与任务可以支撑排障和运维

### P4：流程与设计器能力

优先模块：

1. 表单设计
2. 审批流程设计
3. 打印模板设计
4. 表单数据管理

完成标准：

- 后续财务单据能够基于平台配置生成
- 审批和打印不再依赖硬编码页面结构

## 与后续业务阶段的关系

### 第二阶段：财务管理

在平台一期稳定后进入：

- 报销管理
- 借款管理
- 支付管理
- 凭证管理

### 第三阶段：出纳管理

在财务管理稳定后进入：

- 资金收支
- 银行账户
- 对账管理
- 票据管理
- 出纳台账

## 风险控制

- 不同时改多个前端应用
- 不在平台未稳定时提前开发复杂财务流程
- 不让财务业务直接依赖老系统的任意 SQL 执行模式
- 不把业务模块继续塞进 `sys-*` 目录
